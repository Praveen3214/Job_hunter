4 Hiring Signals New

Find people on LinkedIn who are actively hiring for roles matching your keywords. Two modes: people search and post scanning.

4.1 — People Search (--find-hiring)

Searches LinkedIn People for profiles with the #Hiring badge who match your job keywords.

python main.py --find-hiring -k "Marketing Director,CMO" -l "Bangalore"

What gets extracted

  • Profile info: Name, title, headline, about, location, LinkedIn URL
  • Contact info: Email and phone (from LinkedIn contact overlay, if visible)
  • Company details: Company name, LinkedIn page, website, industry, size
  • Classification: What role they're hiring for (Marketing, Sales, Engineering, Data, etc.)

4.2 — Post Scanning (--find-posts)

Searches LinkedIn content/posts for hiring hashtags and extracts who's posting about open roles.

python main.py --find-posts -k "Marketing Director"

Hashtags scanned (14)

#hiring, #wearehiring, #jobalert, #jobopening, #openposition, #joinourteam, #nowhiring, #hiringalert, #vacancy, #jobopportunity, #recruitment, #careers, #hotjob, #urgentlyhiring

4.3 — Combined Mode

# Both people search + post scan, merged & deduplicated
python main.py --find-hiring --find-posts -k "Head of Marketing"

Results from both sources are merged into a single hiring_signals_*.csv with a signal_source column (people_search or post_search).

5 Consultant Finder New

Discover recruitment consultants, staffing agencies, and headhunters on LinkedIn. Two modes: focused (tied to your keywords) and broad (generic recruiter list).

5.1 — Focused Mode (--find-consultants)

Searches for recruiters who specifically work in your domain by combining recruiter titles with your job keywords.

# Find recruiters who hire for marketing roles in Bangalore
python main.py --find-consultants -k "Marketing Director,CMO" -l "Bangalore"

Builds queries like: "Recruitment Consultant Marketing Director Bangalore", "Headhunter CMO Mumbai"

5.2 — Broad Mode (--consultants-broad)

Searches for ALL recruiters in your target locations, regardless of what roles they fill. Great for building a master recruiter database.

# Find all recruiters in Bangalore + Mumbai (no keyword coupling)
python main.py --consultants-broad -l "Bangalore,Mumbai,Delhi"

Builds queries like: "Recruitment Consultant Bangalore", "Headhunter Mumbai", "Staffing Manager Delhi"

Recruiter titles searched (12)

Recruitment Consultant, Staffing Manager, Headhunter, Executive Search, Placement Consultant, Recruitment Manager, Talent Acquisition Consultant, Recruitment Specialist, Staffing Consultant, Search Consultant, Recruitment Partner, Recruiting

Broad mode uses all 12 titles. Focused mode uses the top 5 to keep query count manageable.

6.2 — First Run (LinkedIn Login)

One-time LinkedIn authentication

The first time you use any LinkedIn-powered feature (--find-hr, --find-hiring, --find-posts, --find-consultants), a Chrome window opens for you to log in manually. Your session is saved in .session/chrome_profile/ so you only need to do this once.

  1. Run any LinkedIn feature (e.g., python main.py --find-hr)
  2. A Chrome window opens at linkedin.com/login
  3. Log in manually (enter your username + password)
  4. Press Enter in the terminal when logged in
  5. Session is saved for all future runs

6.3 — Usage

# During scraping pipeline
python main.py --find-hr

# Use an existing jobs CSV (skip re-scraping)
python main.py --find-hr --jobs-file output/jobs_20260312.csv

9.2 — Common Workflows

Workflow A: Quick LinkedIn-only job scan

python main.py -p linkedin -k "VP Marketing" -l "India" -n 20

Fast scan: ~2 minutes. Good for a quick pulse check on what's available.

Workflow B: "Who's actively hiring?" (people + posts)

python main.py --find-hiring --find-posts -k "Marketing Director,CMO" -l "Bangalore"
--jobs-file output/jobs_LATEST.csv

Skip job scraping, just find people who are actively hiring for marketing roles. Check hiring_signals_*.csv for names, emails, and companies.

Workflow C: Build a recruiter database

python main.py --consultants-broad -l "Bangalore,Mumbai,Delhi" -n 100
--jobs-file output/jobs_LATEST.csv

Builds a master list of all recruitment consultants, staffing firms, and headhunters in your target cities. Check consultants_*.csv for names, agencies, and contact info.

Workflow D: Deep search + HR outreach

# Step 1: Broad job scrape across all platforms
python main.py -n 100

# Step 2: Find HR at those companies + enrich emails
python main.py --find-hr --enrich-emails --jobs-file output/jobs_LATEST.csv

# Step 3: View everything in the dashboard
python serve_dashboard.py

Run in stages to review results between steps and control costs.

Workflow E: Everything at once

python main.py --find-hiring --find-posts --find-consultants --find-hr --enrich-emails --dashboard

The full pipeline: scrape 9 platforms, find hiring signals, discover recruiters, find HR contacts, enrich emails, open dashboard. Takes 30-60 minutes.

10 Troubleshooting

Common issues and how to fix them.

LinkedIn features return 0 results

  • Session may have expired. Delete .session/chrome_profile/ folder and log in again
  • LinkedIn may have flagged your session — wait a few hours before retrying
  • If profiles load but no data is extracted, LinkedIn may have changed its DOM. Check for tool updates

UnicodeEncodeError on Windows

  • This is a Windows console (cp1252) issue with special characters
  • Run with set PYTHONIOENCODING=utf-8 && python main.py
  • Or use Windows Terminal (supports UTF-8 by default) instead of cmd.exe

Naukri / Instahyre / IIMJobs / Hirist returns 0 results

  • Don't close the browser window while scraping
  • Solve any captcha manually if it appears
  • Try reducing -n to 20
  • Ensure Playwright Chromium is installed: playwright install chromium

Hunter.io returns no emails

  • Verify your API key is correct in .env
  • Check your credit balance at hunter.io/dashboard
  • Some small companies don't have domains indexed by Hunter

Dashboard doesn't load CSVs

  • Use the server: python serve_dashboard.py (not direct file:// opening)
  • Or drag-drop CSV files onto the dashboard manually
  • Check that CSV files exist in output/

Scrape Builder shows "A scrape is already running"

  • Only one scrape at a time. Wait for current one to finish
  • If stuck, restart: python serve_dashboard.py

Port already in use

  • Kill the existing process or change port
  • python main.py --dashboard --port 9090

python-jobspy errors / 429 rate-limit

  • Reduce -n to 20-25
  • Search fewer locations at once
  • Wait 10-15 minutes before retrying
⚠
Rate-limiting awareness. All platforms have anti-bot protections. Space out large searches with breaks between runs. LinkedIn-powered features use random delays (3-8 seconds) to stay under the radar.
